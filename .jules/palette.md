## 2024-05-18 - Navigation Header & Sidebar Accessibility Patterns

**Learning:** Found a recurring pattern where top-level structural layout buttons (e.g., Header icons and Sidebar items) frequently lack proper ARIA attributes to define state and semantics. Specifically, the Header's user profile button missed `aria-expanded` and `aria-haspopup`, and the icon-only notification button lacked `aria-label`/`title`. Additionally, the Sidebar's visual indication for active screens lacked the standard programmatic equivalent (`aria-current="page"`). Since these form the foundation of application navigation, these basic attributes are critical for screen readers to convey context and location.
**Action:** When adding or updating layout-level navigation buttons, always verify that icon-only buttons have descriptive `aria-label`s and `title`s. Ensure dropdown toggles include `aria-haspopup="true"` and a dynamically-bound `aria-expanded` attribute. Use the `aria-current="page"` pattern for navigation menu items to denote active status programmatically rather than relying purely on CSS class changes.
